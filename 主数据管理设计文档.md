## 概述

### 功能描述

主数据管理模块整合了商品分类、品牌和材质三大主数据的统一管理,实现了主数据之间的关联关系:

- **商品分类 (Category)**: 商品的层级分类管理
- **品牌 (Brand)**: 关联到商品分类,每个分类可以有多个品牌 (1:n)
- **材质 (Material)**: 关联到商品分类,每个分类可以有多个材质 (1:n)

### 设计目标

1. **数据一致性**: 品牌和材质与分类强关联,避免数据冗余
2. **易用性**: 在商品表单中根据所选分类自动过滤可选的品牌和材质
3. **可维护性**: 集中管理主数据,便于数据维护和审计

## 数据库设计

### ER 图

```
Category (商品分类)
    |
    ├──> Brand (品牌)         [1:n]
    ├──> Material (材质)      [1:n]
    └──> Product (商品)       [1:n]

Product (商品)
    ├──> Category (分类)      [n:1]
    ├──> Brand (品牌)         [n:1]
    └──> Material (材质)      [n:1]
```

### 表结构

#### Category (商品分类表)

| 字段 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | Int | 主键 | PK, AUTO_INCREMENT |
| parentId | Int | 父分类ID | DEFAULT 0 |
| name | String | 分类名称 | NOT NULL |
| sort | Int | 排序 | DEFAULT 0 |
| status | Int | 状态 | DEFAULT 1 |
| createdAt | DateTime | 创建时间 | AUTO |

**关系**:
- 一对多: brands, materials, products

#### Brand (品牌表)

| 字段 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | Int | 主键 | PK, AUTO_INCREMENT |
| name | String | 品牌名称 | NOT NULL |
| categoryId | Int | 所属分类ID | NOT NULL, FK |
| logo | String | 品牌Logo | NULLABLE |
| sort | Int | 排序 | DEFAULT 0 |
| status | Int | 状态 | DEFAULT 1 |
| createdAt | DateTime | 创建时间 | AUTO |
| updatedAt | DateTime | 更新时间 | AUTO |

**索引**:
- UNIQUE(name, categoryId): 同一分类下品牌名称唯一

**关系**:
- 多对一: category (categoryId -> Category.id)
- 一对多: products

#### Material (材质表)

| 字段 | 类型 | 说明 | 约束 |
|------|------|------|------|
| id | Int | 主键 | PK, AUTO_INCREMENT |
| name | String | 材质名称 | NOT NULL |
| categoryId | Int | 所属分类ID | NOT NULL, FK |
| description | String | 材质描述 | NULLABLE |
| sort | Int | 排序 | DEFAULT 0 |
| status | Int | 状态 | DEFAULT 1 |
| createdAt | DateTime | 创建时间 | AUTO |
| updatedAt | DateTime | 更新时间 | AUTO |

**索引**:
- UNIQUE(name, categoryId): 同一分类下材质名称唯一

**关系**:
- 多对一: category (categoryId -> Category.id)
- 一对多: products

#### Product (商品表 - 更新部分)

| 字段 | 类型 | 说明 | 变更 |
|------|------|------|------|
| brandId | Int | 品牌ID | FK -> Brand.id |
| materialId | Int | 材质ID | 🔄 从 `material: String` 改为 `materialId: Int` |

## 后端API设计

### Brand API

**Base URL**: `/api/brands`

#### 获取品牌列表

```http
GET /api/brands?page=1&pageSize=20&categoryId=1&keyword=搜索关键词
```

**响应**:
```json
{
  "code": 200,
  "message": "获取成功",
  "data": {
    "list": [
      {
        "id": 1,
        "name": "周大福",
        "categoryId": 1,
        "category": {
          "id": 1,
          "name": "黄金饰品"
        },
        "logo": "https://...",
        "sort": 0,
        "status": 1,
        "createdAt": "2024-01-01T00:00:00Z"
      }
    ],
    "total": 100,
    "page": 1,
    "pageSize": 20
  }
}
```

#### 创建品牌

```http
POST /api/brands
Content-Type: application/json

{
  "name": "周大福",
  "categoryId": 1,
  "logo": "https://...",
  "sort": 0
}
```

#### 更新品牌

```http
PUT /api/brands/:id
Content-Type: application/json

{
  "name": "周大福",
  "categoryId": 1,
  "logo": "https://...",
  "sort": 0
}
```

#### 删除品牌

```http
DELETE /api/brands/:id
```

**业务规则**:
- 删除前检查是否有商品使用该品牌
- 同一分类下品牌名称不能重复

### Material API

**Base URL**: `/api/materials`

API设计与Brand API类似,支持相同的CRUD操作和过滤参数。

## 功能说明

### 1. 商品分类管理

**功能点**:
- 树形结构展示
- 支持新增、编辑、删除
- 删除时检查是否有关联的品牌、材质或商品

**界面**:
- 使用 Ant Design Tree 组件
- 行内操作按钮(编辑、删除)

### 2. 品牌管理

**功能点**:
- 列表展示,支持分页
- 按分类过滤
- 必须关联到分类
- 同一分类下品牌名称唯一
- 支持上传品牌Logo
- 删除时检查是否有关联商品

**界面**:
- 表格展示品牌列表
- 模态框表单进行新增/编辑
- 分类下拉选择器

### 3. 材质管理

**功能点**:
- 列表展示,支持分页
- 按分类过滤
- 必须关联到分类
- 同一分类下材质名称唯一
- 支持材质描述
- 删除时检查是否有关联商品

**界面**:
- 表格展示材质列表
- 模态框表单进行新增/编辑
- 分类下拉选择器

### 4. 商品表单联动

**功能点**:
- 选择商品分类后,品牌和材质下拉框自动加载该分类下的选项
- 切换分类时,清空已选的品牌和材质
- 未选择分类时,品牌和材质下拉框禁用

**实现逻辑**:
```tsx
// 监听分类变化
const handleCategoryChange = (categoryId: number) => {
  setSelectedCategoryId(categoryId)
  // 清空品牌和材质选择
  form.setFieldsValue({ brandId: undefined, materialId: undefined })
  // 重新加载品牌和材质
  loadBrands(categoryId)
  loadMaterials(categoryId)
}

// 加载品牌
const loadBrands = async (categoryId: number) => {
  const res = await brandApi.getList({ categoryId })
  setBrands(res.data.list)
}
```

## 使用指南

### 管理员操作流程

#### 1. 初始化主数据

**步骤**:
1. 进入"主数据管理"
2. 在"商品分类"标签页,创建分类层级
3. 切换到"品牌管理"标签页,为每个分类添加品牌
4. 切换到"材质管理"标签页,为每个分类添加材质

**示例**:
```
商品分类:
  - 黄金饰品
    - 黄金戒指
    - 黄金项链
  - 银饰
    - 银戒指
    - 银项链

品牌 (黄金饰品):
  - 周大福
  - 老凤祥
  - 六福珠宝

材质 (黄金饰品):
  - 足金999
  - 18K金
  - 22K金
```

#### 2. 添加商品

**步骤**:
1. 进入"商品管理"
2. 点击"新增商品"
3. 选择商品分类
4. 从自动过滤的下拉框中选择品牌和材质
5. 填写其他信息并保存

### 数据维护

#### 修改品牌/材质关联的分类

**影响**:
- 已关联该品牌/材质的商品不受影响
- 新增商品时,只能在新分类下选择该品牌/材质

**建议**:
- 谨慎修改分类关联
- 如需调整,建议新建品牌/材质记录

#### 删除品牌/材质

**前置检查**:
- 系统会检查是否有商品使用该品牌/材质
- 如有关联商品,不允许删除
- 需先将商品的品牌/材质改为其他值或清空

## 数据迁移说明

### 从旧版本升级

如果从 v1.0.0 (material字段为String) 升级到 v1.1.0 (materialId为Int):

**迁移脚本**:
```sql
-- 1. 为每个分类创建常用材质
INSERT INTO Material (name, categoryId, sort) VALUES
  ('纯银', 1, 0),
  ('镀金', 1, 1),
  ('不锈钢', 1, 2);

-- 2. 更新Product表的materialId
UPDATE Product 
SET materialId = (SELECT id FROM Material WHERE name = Product.material AND categoryId = Product.categoryId LIMIT 1)
WHERE material IS NOT NULL;

-- 3. (可选) 清理旧的material字段
-- ALTER TABLE Product DROP COLUMN material;
```

## 技术要点

### 1. 级联查询优化

```typescript
// 后端: 使用Prisma include
const brands = await prisma.brand.findMany({
  include: {
    category: {
      select: { id: true, name: true }
    }
  }
})
```

### 2. 前端缓存策略

```typescript
// 商品表单中缓存分类对应的品牌和材质
const [brandsByCategory, setBrandsByCategory] = useState<Record<number, any[]>>({})

const loadBrands = async (categoryId: number) => {
  if (brandsByCategory[categoryId]) {
    setBrands(brandsByCategory[categoryId])
    return
  }
  const res = await brandApi.getList({ categoryId })
  setBrandsByCategory(prev => ({ ...prev, [categoryId]: res.data.list }))
  setBrands(res.data.list)
}
```

### 3. 唯一约束

```prisma
model Brand {
  @@unique([name, categoryId])
}

model Material {
  @@unique([name, categoryId])
}
```
