# 主数据管理功能实现总结

> 实施日期: 2026-05-28
> 状态: ✅ 完成

## 📋 实现概览

将原有的"分类管理"页面升级为"主数据管理"页面,整合了商品分类、品牌和材质三大主数据的统一管理,建立了分类与品牌、材质的 1:n 关联关系。

- ✅ 材质字段从 Input 改为 Select 下拉框

- ✅ 切换分类时,自动清空已选的品牌和材质

**表单字段变更**:
```tsx
// 旧版本
<Form.Item name="brand" label="品牌">
  <Input placeholder="请输入品牌" />
</Form.Item>

<Form.Item name="material" label="材质">
  <Input placeholder="如: 纯银、镀金、不锈钢" />
</Form.Item>

// 新版本
<Form.Item name="brandId" label="品牌">
  <Select
    placeholder="请先选择分类"
    disabled={!selectedCategoryId}
    options={brands.map(b => ({ label: b.name, value: b.id }))}
    allowClear
  />
</Form.Item>

<Form.Item name="materialId" label="材质">
  <Select
    placeholder="请先选择分类"
    disabled={!selectedCategoryId}
    options={materials.map(m => ({ label: m.name, value: m.id }))}
    allowClear
  />
</Form.Item>
```

## 🔄 数据迁移步骤

如果系统已有旧数据,需要执行以下迁移:

### 1. 备份数据库

```bash
cp backend/data/shop.db backend/data/shop.db.backup
```

### 2. 执行Schema更新

```bash
cd backend
npx prisma db push
```

### 3. 数据迁移 (如果Product表中有material字符串数据)

需要手动创建迁移脚本或通过管理界面:
1. 为每个分类创建对应的材质记录
2. 更新Product表的materialId字段
3. (可选) 清理旧的material字段

## 📝 使用示例

### 1. 初始化主数据

```
进入系统 -> 主数据管理

步骤1: 商品分类标签页
  创建分类: "黄金饰品" -> "黄金戒指"
  
步骤2: 品牌管理标签页
  新增品牌: "周大福", 所属分类: "黄金饰品"
  新增品牌: "老凤祥", 所属分类: "黄金饰品"
  
步骤3: 材质管理标签页
  新增材质: "足金999", 所属分类: "黄金饰品"
  新增材质: "18K金", 所属分类: "黄金饰品"
```

### 2. 添加商品

```
进入系统 -> 商品管理 -> 新增商品

步骤1: 选择分类 -> "黄金戒指"
步骤2: 品牌下拉框自动加载 -> 选择"周大福"
步骤3: 材质下拉框自动加载 -> 选择"足金999"
步骤4: 填写其他信息 -> 保存
```
