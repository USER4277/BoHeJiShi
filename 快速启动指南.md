
**执行内容**:

## 二、管理命令

### 查看服务状态

```bash
./status.sh
```

**显示内容**:
- 后端服务状态（运行/停止、PID、端口、健康检查）
- 前端服务状态（运行/停止、PID、端口、健康检查）
- 数据库文件状态（大小、路径）
- 日志文件信息（大小、行数）

---

### 停止服务

```bash
./stop.sh
```

**执行内容**:
- 停止后端服务（先尝试优雅停止，5秒后强制停止）
- 停止前端服务（先尝试优雅停止，5秒后强制停止）
- 清理端口占用（3001、5173）
- 删除 PID 文件

---

### 重启服务

```bash
./restart.sh
```

**执行内容**:
- 调用 `stop.sh` 停止服务
- 等待 3 秒
- 调用 `start.sh` 启动服务

## 五、数据备份

### 数据库位置

```
~/Documents/薄荷集市/data/shop.db
```

### 自动备份

- **时间**: 每日 23:00
- **位置**: `~/Documents/薄荷集市/data/backup/`
- **保留**: 30 天
- **格式**: `shop_backup_YYYY-MM-DD_HH-mm-ss.db`

### 手动备份

```bash
cd backend
npx tsx -e "
import { backupDatabase } from './src/utils/backup';
backupDatabase().then(f => console.log('备份成功:', f));
"
```

### 恢复备份

```bash
# 1. 停止服务
./stop.sh

# 2. 备份当前数据库
cp ~/Documents/薄荷集市/data/shop.db ~/Documents/薄荷集市/data/shop.db.bak

# 3. 恢复指定备份文件
cp ~/Documents/薄荷集市/data/backup/shop_backup_2026-05-27_23-00-00.db \
   ~/Documents/薄荷集市/data/shop.db

# 4. 启动服务
./start.sh
```

## 六、常见问题

### Q1: 端口被占用

**现象**:
```
Error: listen EADDRINUSE: address already in use :::3001
```

**解决方法**:
```bash
# 方法1: 使用 stop.sh（推荐）
./stop.sh

# 方法2: 手动查找并停止
lsof -i :3001
kill -9 <PID>
```

---

### Q2: Node.js 版本过低

**现象**:
```
✗ Node.js 版本过低 (需要 >= 18)
```

**解决方法**:
```bash
# 使用 nvm 安装 Node.js 20
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 20
nvm use 20
```

---

### Q3: 数据库初始化失败

**解决方法**:
```bash
cd backend
npx prisma generate
npx prisma db push
npx tsx src/scripts/init-admin.ts
```

---

### Q4: 前端白屏/无法访问

**排查步骤**:
```bash
# 1. 检查服务状态
./status.sh

# 2. 查看前端日志
tail -f logs/frontend.log

# 3. 检查后端API
curl http://localhost:3001/api/auth/me -u admin:admin123

# 4. 重启服务
./restart.sh
```

---

### Q5: 忘记管理员密码

**解决方法**:
```bash
cd backend
npx tsx src/scripts/init-admin.ts
```

**提示**: 会覆盖现有 admin 账户，密码重置为 `admin123`

## 八、生产部署

### 构建前端

```bash
cd frontend
npm run build
```

输出目录: `frontend/dist/`

### 构建后端

```bash
cd backend
npm run build
```

输出目录: `backend/dist/`

### 生产环境运行

```bash
# 使用 PM2（推荐）
npm install -g pm2

# 启动后端
cd backend
pm2 start dist/index.js --name bohejishi-backend

# 启动前端（使用静态服务器）
pm2 serve frontend/dist 5173 --spa --name bohejishi-frontend

# 查看状态
pm2 status

# 查看日志
pm2 logs
```
