## 1. 文档概述

### 1.1 项目背景

薄荷集市是一家专注于时尚饰品销售的品牌店铺，主营产品包括戒指、项链、手链、耳饰、发饰、胸针等各类时尚配饰。随着业务规模不断扩大，传统的纸质记账和Excel表格管理已无法满足日常运营需求，亟需一套完整的店铺管理系统来提升运营效率、降低管理成本、实现数字化转型。

### 1.2 系统目标

本系统旨在为薄荷集市打造一套高效、稳定、易用的店铺管理系统，实现以下核心目标：

- **业务数字化**：将商品管理、库存管理、销售管理、财务管理等核心业务环节全部纳入系统管理
- **流程自动化**：通过系统自动化处理日常业务，减少人工操作失误，提高工作效率
- **决策数据化**：提供多维度数据分析和可视化报表，为经营决策提供数据支撑
- **服务标准化**：建立统一的客户服务标准，提升客户满意度和忠诚度

### 1.3 适用范围

本文档适用于以下对象：

- 项目开发团队：作为系统开发的依据
- 产品经理：作为需求确认和变更的基准
- 测试团队：作为测试用例编写的参考
- 业务部门：作为业务流程培训的教材
- 运维团队：作为系统部署和维护的指南

### 1.4 名词定义

| 术语 | 定义 |
|------|------|
| SKU | Stock Keeping Unit，库存量单位，即库存控制的最小可用单位 |
| POS | Point of Sale，销售终端，指收银系统 |
| CRM | Customer Relationship Management，客户关系管理 |
| ERP | Enterprise Resource Planning，企业资源计划 |
| MVP | Minimum Viable Product，最小可行产品 |
| API | Application Programming Interface，应用程序接口 |

### 3.2 系统管理模块 (SYS)

#### 3.2.1 功能概述

系统管理模块是整个系统的基础模块，提供用户管理、角色权限管理、系统参数配置、操作日志、消息通知等功能，确保系统安全稳定运行。

#### 3.2.2 功能详细说明

##### 3.2.2.1 用户管理

| 功能点 | 功能描述 | 操作流程 |
|--------|----------|----------|
| 用户列表 | 显示所有系统用户，支持按姓名、手机号模糊查询 | 查询→筛选→列表展示 |
| 新增用户 | 添加新用户，录入基本信息、分配角色 | 新增→填写信息→选择角色→保存 |
| 编辑用户 | 修改用户基本信息 | 查询→编辑→修改信息→保存 |
| 重置密码 | 重置用户登录密码 | 查询→重置→确认→发送新密码 |
| 启用/禁用 | 启用或禁用用户账号 | 查询→状态切换→确认 |

**用户信息字段**：

| 字段名称 | 字段类型 | 必填 | 说明 |
|----------|----------|------|------|
| 用户编号 | varchar(20) | 是 | 系统自动生成，格式：USER+6位序号 |
| 用户姓名 | varchar(50) | 是 | - |
| 手机号码 | varchar(20) | 是 | 用于登录和接收通知 |
| 邮箱 | varchar(100) | 否 | - |
| 所属部门 | varchar(50) | 是 | - |
| 岗位 | varchar(50) | 是 | - |
| 角色 | varchar(50) | 是 | 关联角色权限 |
| 状态 | int | 是 | 0-禁用，1-启用 |
| 创建时间 | datetime | 是 | 自动生成 |

##### 3.2.2.2 角色权限管理

| 功能点 | 功能描述 |
|--------|----------|
| 角色列表 | 显示所有角色，支持新增、编辑、删除角色 |
| 角色配置 | 配置角色对应的功能权限和数据权限 |
| 功能权限 | 按模块配置角色可访问的功能菜单 |
| 数据权限 | 配置角色可访问的数据范围（如：只看本店铺数据） |

**预置角色**：

| 角色代码 | 角色名称 | 权限说明 |
|----------|----------|----------|
| ADMIN | 超级管理员 | 拥有系统所有权限，可进行所有操作 |

|--------|----------|----------|
| 分类列表 | 显示商品分类树形结构，支持增删改 | 查询→展示树形结构 |

### 3.4 库存管理模块 (INV)

#### 3.4.1 功能概述

库存管理模块负责管理店铺库存，实现库存的入库、出库、盘点、预警等功能，确保库存数据准确，避免缺货或积压。

#### 3.4.2 功能详细说明

##### 3.4.2.1 库存查询

| 功能点 | 功能描述 | 操作流程 |
|--------|----------|----------|
| 实时库存 | 查询当前各商品的库存数量 | 筛选条件→查询→展示库存列表 |
| 库存明细 | 查看某商品库存的详细来源 | 选择商品→查看明细 |
| 库龄分析 | 分析商品在库时长 | 按时间范围→统计库龄分布 |

**库存查询条件**：

| 查询项 | 说明 |
|--------|------|
| 仓库 | 支持多仓库筛选 |
| 商品分类 | 按商品分类筛选 |
| 商品编码/名称 | 关键字搜索 |
| 库存状态 | 正常/预警/缺货 |

**库存列表字段**：

| 字段名称 | 说明 |
|----------|------|
| 商品编码 | - |
| 商品名称 | - |
| 规格属性 | SKU规格组合 |
| 库存数量 | 当前库存 |
| 安全库存 | 设置的安全库存值 |
| 预警状态 | 正常/预警/缺货 |
| 最近入库时间 | - |
| 最近出库时间 | - |

##### 3.4.2.2 入库管理

| 功能点 | 功能描述 | 操作流程 |
|--------|----------|----------|
| 采购入库 | 根据采购单入库 | 选择采购单→验收→入库确认 |
| 退货入库 | 客户退货商品入库 | 录入退货信息→验收→入库 |
| 调拨入库 | 从其他仓库调入 | 调拨单确认→入库 |
| 其他入库 | 其他类型的入库 | 填写入库单→审批→入库 |

**入库单字段**：

| 字段名称 | 类型 | 说明 |
|----------|------|------|
| 入库单号 | varchar(20) | 自动生成，格式：RK+日期+序号 |
| 入库类型 | int | 1-采购入库，2-退货入库，3-调拨入库，4-其他 |
| 供应商 | int | 关联供应商 |
| 入库仓库 | int | 关联仓库 |
| 经办人 | varchar(50) | - |
