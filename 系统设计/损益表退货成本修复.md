# 损益表退货成本问题修复

## 问题描述

损益表在计算毛利润时，存在一个严重的会计逻辑错误：

**问题症状：** 当有退货发生时，报表显示虚假的亏损状况。

**问题根源：**
1. 收入端：正确扣除了退货金额（净收入 = 销售收入 - 退货金额）
2. 成本端：**错误地保留了退货商品的成本**
3. 结果：毛利润 = 净收入 - 包含退货的成本 → 利润被错误地压低

## 示例说明

### 场景
- 销售 3 件商品，每件售价 100 元，成本 60 元
- 退货 1 件，退款 100 元

### 修复前（错误）
```
销售收入：300 元
退货金额：-100 元
净收入：200 元

销售成本：180 元（3 件商品的成本，包含退货的那件）
毛利润：200 - 180 = 20 元
毛利率：10%

问题：实际上我们只卖出了 2 件商品，成本应该是 120 元，
     但我们还在计算退回商品的 60 元成本！
```

### 修复后（正确）
```
销售收入：300 元
退货金额：-100 元
净收入：200 元

销售成本：180 元
退货成本：-60 元（退回商品的成本需要扣除）
净销售成本：120 元
毛利润：200 - 120 = 80 元
毛利率：40%

正确：实际利润 = 每件 40 元利润 × 2 件 = 80 元
```

## 技术实现

### 后端修改 (`/backend/src/routes/financial.ts`)

#### 修改前
```javascript
// 只查询退货金额
const returns = await prisma.orderReturn.aggregate({
  where: { createdAt: { gte: start, lte: end }, status: { in: [1, 2] } },
  _sum: { returnAmount: true },
  _count: true
});

// 计算时使用全部销售成本
const costOfSales = parseFloat(totalCost.toFixed(2));
const grossProfit = netRevenue - costOfSales;  // ❌ 错误
```

#### 修改后
```javascript
// 查询退货详情，包含订单明细
const returns = await prisma.orderReturn.findMany({
  where: { createdAt: { gte: start, lte: end }, status: { in: [1, 2] } },
  include: {
    order: {
      include: {
        items: {
          include: {
            sku: { include: { product: true } }
          }
        }
      }
    }
  }
});

// 计算退货成本
let returnCost = 0;
let returnAmount = 0;
returns.forEach(returnOrder => {
  returnAmount += returnOrder.returnAmount;
  // 累加该退货订单所有商品的成本
  returnOrder.order.items.forEach(item => {
    const costPrice = item.sku.product.costPrice || 0;
    returnCost += costPrice * item.quantity;
  });
});

// 净销售成本 = 总成本 - 退货成本
const netCostOfSales = parseFloat((totalCost - returnCost).toFixed(2));
const costOfSales = netCostOfSales;
const grossProfit = netRevenue - costOfSales;  // ✓ 正确
```

## 影响范围

### 受影响的指标
1. **销售成本** - 现在正确扣除退货成本
2. **毛利润** - 准确反映实际盈利
3. **毛利率** - 正确的百分比
4. **营业利润** - 基于正确的毛利润计算
5. **净利润** - 最终利润数据准确

**响应数据新增字段：**
```json

  "cost": {
    "costOfSales": 120.00,      // 净销售成本（已扣除退货）
    "returnCost": 60.00,        // 新增：退货成本
    "grossProfit": 80.00,       // 修正后的毛利润
    "grossProfitMargin": 40.00  // 修正后的毛利率
  }
}
```

## 会计原则

这个修复遵循了基本的会计配比原则（Matching Principle）：

> **收入与成本必须配比**
> - 确认的收入应当与其对应的成本配比
> - 退货时，既要冲减收入，也要冲减对应的成本
> - 只有真正售出的商品成本才应计入销售成本

## 测试验证

可以通过以下场景验证修复：

1. 创建订单：销售商品
2. 查看损益表：记录初始利润
3. 退货：部分或全部退货
4. 再次查看损益表：
   - ✓ 收入减少
   - ✓ 成本也应该减少
   - ✓ 毛利润应该反映实际情况，不应该虚假亏损
